📌 Python Getter & Setter (Controlled Access)

1. What are Getters and Setters?

Getters and setters provide controlled access to private variables.

  • Getter → Reads private data
  • Setter → Updates private data with validation

2. Getter & Setter Example 

class Person:

    def __init__(self, age):

        self.__age = age


    def get_age(self):

        return self.__age


    def set_age(self, age):

        if age > 0:

            self.__age = age

Usage: 

p = Person(30)

print(p.get_age())

p.set_age(40)

3. Using @property (Modern Python Way) 

class Person:

    def __init__(self, age):

        self.__age = age


    @property
    def age(self):

        return self.__age


    @age.setter
    def age(self, value):

        if value > 0:

            self.__age = value


p = Person(30)

print(p.age)

# getter

p.age = 40

# setter

4. How @property Works

  • @property converts method into getter property
  • @age.setter creates setter for same property
  • Allows method access like normal variable

5. Benefits of Getter & Setter

Benefit Explanation
Data protection Prevent invalid modifications
Validation Control allowed input values
Maintainability Internal implementation remains hidden

Abstraction is not only about missing method definition.

It is about:

  • Defining a contract/structure

Abstraction means defining methods without implementation inside parent abstract class.

    @abstractmethod

class Dog(Animal):

        return "Bark"

📌 Pass by Reference vs Pass by Value (Python Object References)

1. Pass by Value

A full copy of the data is sent to the function.

Changes inside the function do NOT affect the original value.

2. Pass by Reference

A direct reference to the original memory location is passed.

Changes inside the function affect the original object.

3. Python's Actual Behavior

Python uses: 

Pass by Object Reference

The reference itself is passed by value.

Both variables point to the same object in memory.

📌 GIL (Global Interpreter Lock) in Python

1. What is GIL?

GIL stands for: 

Global Interpreter Lock

It is a lock in CPython that allows:

Only ONE thread to execute Python bytecode at a time per process.

2. Main Problem with GIL

Python has:

  • ✅ GIL (Global Interpreter Lock)
  • Only ONE thread executes Python bytecode at a time
  • Threads are not ideal for CPU-heavy tasks

Python rapidly switches between threads, but they do not execute Python CPU instructions truly in parallel.

3. Important Interview Confusion

GIL ≠ Single-threaded program

Python programs can still:

  • Create multiple threads
  • Switch between them
  • Run concurrently

But:

Only one thread executes Python bytecode at once.

4. Parallelism vs Concurrency

Parallelism Tasks run simultaneously
